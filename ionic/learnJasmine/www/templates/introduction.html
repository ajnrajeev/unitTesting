Introduction

Design Considerations Separation of concerns

AngularJS architecture is based on micro-services and a module(controller/service/filter/directive) or a function is defined to accomplish a very specific task.

So a controller scope contains only variables and functions specifically required for DOM manipulation.

If data is stored in a server, the correspoding AJAX request to fetch the required data is implemented as a service and the dependency injected into the required module

 Dependency Injection

AngularJS dependency injection which is a direct result of its micro-service architecture is useful in isolating the specific piece of code under test by mocking the injected dependencies

This facilitates testing of a service
