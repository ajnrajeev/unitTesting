Jasmine Basics

Test Suite Test suites in Jasmine are defined using describe
describe("Test something", function(){})
 Test Spec Test specs start with it.
it("should test something and output this", function(){})
 Expectations Expectations are set and matched with the required result to define PASS/FAIL. This is done using expect. The matchers can be for a positive result or a negative result, as follows. 

expect(expression).toBe(expectedResult)

expect(expression).not.toBe(notExpectedExpressions)

 Disabling suites This is achieved by pre-fixing the suite with an x. So any spec defined as xdescribe is not taken up for execution. Disabling Specs Test specs can be disabled by
prefixing the definition with an x,
defining the spec with no function argument, or
calling pending(reasonString) within the spec.
 Spies Spies are used to mock any custom services.
